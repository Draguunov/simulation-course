## Реализация

### Общий подход

Для каждого моделирования используется **базовый датчик** – в данной работе применяется встроенный генератор Python `random.random()`, возвращающий равномерно распределённое число в интервале [0, 1). Однако алгоритм выбора события **самописный** и основан на методе, описанном в лекции:

1. Задаётся полная группа попарно несовместных событий с вероятностями \(p_1, p_2, \dots, p_m\), где \(\sum_{i=1}^m p_i = 1\).
2. Генерируется \(\alpha \in [0, 1)\).
3. Последовательно проверяется:
   - если \(\alpha \le p_1\), происходит событие 1;
   - иначе \(\alpha := \alpha - p_1\) и проверяется условие \(\alpha \le p_2\);
   - и так далее, пока не будет выбрано событие.

Этот алгоритм гарантирует, что событие с вероятностью \(p_k\) будет выбрано с частотой, стремящейся к \(p_k\) при большом числе испытаний.

### Приложение «Скажи “да” или “нет”»

- **События:**  
  \(A_1\) – ответ «Да», вероятность \(p\) (задаётся пользователем);  
  \(A_2\) – ответ «Нет», вероятность \(1-p\).
- **Алгоритм:**  
  Генерируется \(\alpha\). Если \(\alpha \le p\), ответ «Да», иначе – «Нет».
- **Интерфейс:**  
  Поле ввода вопроса, поле для вероятности «Да», кнопка «Получить ответ». Выводится вопрос и ответ. Пояснение алгоритма отображается внизу.

### Приложение «Шар предсказаний»

- **События:**  
  8 предсказаний с заданными вероятностями (сумма = 1):
  - Бесспорно – 0.15  
  - Вероятнее всего – 0.15  
  - Хорошие перспективы – 0.15  
  - Пока не ясно – 0.20  
  - Спроси позже – 0.10  
  - Сомнительно – 0.10  
  - Мой ответ: нет – 0.10  
  - Очень маловероятно – 0.05
- **Алгоритм:**  
  Генерируется \(\alpha\), затем последовательно вычитаются вероятности до тех пор, пока \(\alpha\) не станет меньше или равна очередной вероятности. Выбранное предсказание возвращается.
- **Интерфейс:**  
  Поле ввода вопроса, кнопка «Предсказать», вывод предсказания. Статистика (частота и относительная частота каждого исхода) отображается в таблице. Кнопка «Очистить статистику» сбрасывает счётчики.

### Особенности

- Все вероятности задаются явно (вручную). Для «Шара предсказаний» последняя вероятность вычисляется как \(1 - \sum\) предыдущих, что гарантирует полноту группы.
- Алгоритм выбора события использует **только** базовый датчик и операции вычитания – нет вызовов `random.choice` или других встроенных средств выбора по вероятности.
- Интерфейс построен на `tkinter` и `ttk`, предоставляет удобное управление и наглядную статистику.

## Теоретическая часть

В стохастическом моделировании случайные события описываются с помощью **базового датчика случайных чисел**, который генерирует случайную величину

\[
\alpha \in [0,1).
\]

Если событие \(A\) имеет вероятность \(P(A)=p\), то оно моделируется следующим образом:

- генерируется случайное число \(\alpha\);
- если \(\alpha < p\), то событие \(A\) считается произошедшим.

Следовательно,

\[
P\{\alpha < p\} = p.
\]

Если необходимо смоделировать одно событие из **полной группы попарно несовместных событий**
\[
A_1, A_2, \dots, A_m,
\]
для которых заданы вероятности
\[
P(A_i)=p_i, \qquad \sum_{i=1}^{m} p_i = 1,
\]
то используется следующий алгоритм:

1. Генерируется случайное число \(\alpha \in [0,1)\);
2. Из \(\alpha\) последовательно вычитаются вероятности \(p_1, p_2, \dots\);
3. На том шаге, когда \(\alpha \le 0\), считается, что произошло соответствующее событие.

Такой подход эквивалентен разбиению отрезка \([0,1)\) на интервалы длиной \(p_i\), каждому из которых соответствует своё событие.

Для корректного моделирования группа событий должна быть:

- **полной** — одно из событий обязательно должно произойти;
- **попарно несовместной** — наступление одного события исключает наступление другого.

Если группа событий неполная или события совместны, её необходимо преобразовать к полной группе несовместных исходов.

## Реализация

### Общий подход

Для моделирования используется **базовый датчик** — встроенный генератор Python `random.random()`, возвращающий равномерно распределённое число на интервале \([0,1)\).

При этом алгоритм выбора события реализован **самостоятельно** и основан на методе, рассмотренном на лекции:

1. задаётся полная группа событий с вероятностями \(p_1, p_2, \dots, p_m\);
2. генерируется \(\alpha \in [0,1)\);
3. вероятности последовательно вычитаются из \(\alpha\);
4. на шаге, где \(\alpha \le 0\), выбирается соответствующее событие.

Таким образом, в программе не используются `random.choice()` или другие встроенные механизмы выбора события по вероятности.

### Приложение «Скажи “да” или “нет”»

В первой части моделируется случайное событие с двумя исходами:

- \(A_1\) — ответ **«Да»**, вероятность \(p\);
- \(A_2\) — ответ **«Нет»**, вероятность \(1-p\).

Алгоритм работы:

1. пользователь вводит вопрос;
2. задаётся вероятность ответа «Да»;
3. генерируется случайное число \(\alpha\);
4. из \(\alpha\) последовательно вычитаются вероятности событий;
5. определяется, какое событие произошло.

**Интерфейс приложения** включает:

- поле ввода вопроса;
- поле для вероятности ответа «Да»;
- кнопку «Получить ответ»;
- вывод вопроса и результата;
- краткое пояснение алгоритма.

### Приложение «Шар предсказаний»

Во второй части моделируется выбор одного исхода из полной группы событий — возможных предсказаний.

Используются 8 ответов:

- Бесспорно  
- Вероятнее всего  
- Хорошие перспективы  
- Пока не ясно  
- Спроси позже  
- Сомнительно  
- Мой ответ: нет  
- Очень маловероятно  

Для первых событий вероятности задаются вручную, а последняя вероятность вычисляется автоматически как остаток до единицы:

\[
p_m = 1 - \sum_{i=1}^{m-1} p_i.
\]

Это гарантирует выполнение условия

\[
\sum_{i=1}^{m} p_i = 1.
\]

Алгоритм работы:

1. пользователь вводит вопрос;
2. задаётся список событий и их вероятностей;
3. генерируется случайное число \(\alpha\);
4. из \(\alpha\) последовательно вычитаются вероятности событий;
5. на шаге, где \(\alpha \le 0\), выбирается соответствующее предсказание;
6. обновляется статистика результатов.

**Интерфейс приложения** включает:

- поле ввода вопроса;
- кнопку «Предсказать»;
- вывод предсказания;
- таблицу статистики, содержащую:
  - теоретическую вероятность;
  - частоту появления;
  - относительную частоту;
- кнопку «Очистить статистику».

### Особенности реализации

- все вероятности задаются явно;
- для «Шара предсказаний» последняя вероятность вычисляется автоматически как остаток до 1;
- выбор события выполняется только через базовый датчик и последовательное вычитание вероятностей;
- интерфейс реализован с использованием `tkinter` и `ttk`;
- в приложении предусмотрена статистическая обработка результатов.
