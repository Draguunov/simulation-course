## Реализация

### Общий подход

Для каждого моделирования используется **базовый датчик** – в данной работе применяется встроенный генератор Python `random.random()`, возвращающий равномерно распределённое число в интервале [0, 1). Однако алгоритм выбора события **самописный** и основан на методе, описанном в лекции:

1. Задаётся полная группа попарно несовместных событий с вероятностями \(p_1, p_2, \dots, p_m\), где \(\sum_{i=1}^m p_i = 1\).
2. Генерируется \(\alpha \in [0, 1)\).
3. Последовательно проверяется:
   - если \(\alpha \le p_1\), происходит событие 1;
   - иначе \(\alpha := \alpha - p_1\) и проверяется условие \(\alpha \le p_2\);
   - и так далее, пока не будет выбрано событие.

Этот алгоритм гарантирует, что событие с вероятностью \(p_k\) будет выбрано с частотой, стремящейся к \(p_k\) при большом числе испытаний.

### Приложение «Скажи “да” или “нет”»

- **События:**  
  \(A_1\) – ответ «Да», вероятность \(p\) (задаётся пользователем);  
  \(A_2\) – ответ «Нет», вероятность \(1-p\).
- **Алгоритм:**  
  Генерируется \(\alpha\). Если \(\alpha \le p\), ответ «Да», иначе – «Нет».
- **Интерфейс:**  
  Поле ввода вопроса, поле для вероятности «Да», кнопка «Получить ответ». Выводится вопрос и ответ. Пояснение алгоритма отображается внизу.

### Приложение «Шар предсказаний»

- **События:**  
  8 предсказаний с заданными вероятностями (сумма = 1):
  - Бесспорно – 0.15  
  - Вероятнее всего – 0.15  
  - Хорошие перспективы – 0.15  
  - Пока не ясно – 0.20  
  - Спроси позже – 0.10  
  - Сомнительно – 0.10  
  - Мой ответ: нет – 0.10  
  - Очень маловероятно – 0.05
- **Алгоритм:**  
  Генерируется \(\alpha\), затем последовательно вычитаются вероятности до тех пор, пока \(\alpha\) не станет меньше или равна очередной вероятности. Выбранное предсказание возвращается.
- **Интерфейс:**  
  Поле ввода вопроса, кнопка «Предсказать», вывод предсказания. Статистика (частота и относительная частота каждого исхода) отображается в таблице. Кнопка «Очистить статистику» сбрасывает счётчики.

### Особенности

- Все вероятности задаются явно (вручную). Для «Шара предсказаний» последняя вероятность вычисляется как \(1 - \sum\) предыдущих, что гарантирует полноту группы.
- Алгоритм выбора события использует **только** базовый датчик и операции вычитания – нет вызовов `random.choice` или других встроенных средств выбора по вероятности.
- Интерфейс построен на `tkinter` и `ttk`, предоставляет удобное управление и наглядную статистику.

## Теоретическая часть

В стохастическом моделировании случайные события удобно описывать через базовый датчик случайных чисел, который генерирует случайную величину

\[
\alpha \in [0,1).
\]

Если событие \(A\) имеет вероятность

\[
P(A) = p,
\]

то оно может быть смоделировано следующим образом:

- генерируется случайное число \(\alpha\);
- если \(\alpha < p\), то событие \(A\) считается произошедшим.

Следовательно,

\[
P\{\alpha < p\} = p.
\]

Такой подход лежит в основе моделирования простого случайного события. :contentReference[oaicite:1]{index=1}

Если необходимо смоделировать одно событие из **полной группы попарно несовместных событий**
\[
A_1, A_2, \dots, A_m,
\]
для которых заданы вероятности
\[
P(A_i) = p_i, \qquad \sum_{i=1}^{m} p_i = 1,
\]
то используется следующий алгоритм:

1. Генерируется случайное число \(\alpha \in [0,1)\);
2. Из \(\alpha\) последовательно вычитаются вероятности \(p_1, p_2, \dots\);
3. На том шаге, когда значение становится меньше либо равно нулю, считается, что произошло соответствующее событие.

Такой метод эквивалентен разбиению интервала \([0,1)\) на подынтервалы длиной \(p_i\), каждому из которых соответствует своё событие. :contentReference[oaicite:2]{index=2}

Для корректного моделирования группа событий должна быть:

- **полной** — одно из событий обязательно должно произойти;
- **попарно несовместной** — наступление одного события исключает наступление другого.

Если группа событий неполная или события совместны, необходимо преобразовать исходную постановку к полной группе несовместных исходов. Это также отмечается в лекционном материале. :contentReference[oaicite:3]{index=3}

## Описание разработанного приложения

В ходе работы было создано GUI-приложение на языке Python с использованием библиотеки `tkinter`.

Приложение состоит из двух частей:

### Часть 1. Приложение «Скажи “да” или “нет”»

В первой части моделируется простое случайное событие с двумя исходами:

- **«Да»** с вероятностью \(p = 0.5\);
- **«Нет»** с вероятностью \(1 - p = 0.5\).

Для получения результата используется случайная величина \(\alpha\), генерируемая встроенным базовым датчиком Python `random.random()`. После этого выполняется последовательное вычитание вероятностей:

- если после вычитания вероятности события «Да» значение становится меньше либо равно нулю, выбирается ответ **«Да»**;
- иначе выбирается ответ **«Нет»**.

Таким образом реализуется модель случайного события в соответствии с лекцией.

### Часть 2. Приложение «Шар предсказаний» (Magic 8-Ball)

Во второй части моделируется выбор одного исхода из группы событий — возможных предсказаний шара.

Для каждого ответа задаётся вероятность. Первые вероятности задаются вручную в коде, а последняя вычисляется автоматически как остаток до единицы:

\[
p_m = 1 - \sum_{i=1}^{m-1} p_i.
\]

Это позволяет гарантировать, что сумма вероятностей всех исходов равна единице, а сами ответы образуют полную группу событий.

После генерации случайной величины \(\alpha\) алгоритм снова использует последовательное вычитание вероятностей. На шаге, где \(\alpha \le 0\), выбирается соответствующее предсказание.

Кроме того, в приложении ведётся статистика:

- частота появления каждого ответа;
- относительная частота каждого ответа.

Это позволяет наблюдать, как эмпирические вероятности приближаются к заданным теоретическим значениям.

## Алгоритм работы программы

### Для приложения «Да / Нет»

1. Пользователь вводит вопрос;
2. Генерируется случайное число \(\alpha\);
3. Формируется группа событий:
   - \(A_1 =\) «Да», \(p_1 = 0.5\);
   - \(A_2 =\) «Нет», \(p_2 = 0.5\);
4. Из \(\alpha\) последовательно вычитаются вероятности;
5. Определяется произошедшее событие;
6. Ответ выводится на экран.

### Для приложения «Шар предсказаний»

1. Пользователь вводит вопрос;
2. Задаётся список возможных ответов и их вероятностей;
3. Последняя вероятность вычисляется как остаток до 1;
4. Генерируется случайное число \(\alpha\);
5. Из \(\alpha\) последовательно вычитаются вероятности ответов;
6. На шаге, где \(\alpha \le 0\), выбирается соответствующий ответ;
7. Обновляется статистика появления событий;
8. Результат и статистика отображаются в интерфейсе.
