## Теоретическая справка

### Цепь Маркова с непрерывным временем
- Состояния: 1 – Ясно, 2 – Облачно, 3 – Пасмурно.
- Переходы происходят в случайные моменты времени.
- Интенсивность перехода из состояния $\(i\)$ в состояние $\(j\)$ ( $\(i \ne j\)$ ) обозначается $\(\lambda_{ij}\$).
- Полная интенсивность выхода из состояния \(i\):
  $\[
  \lambda_i = \sum_{j \ne i} \lambda_{ij}
  \]$
- Время пребывания в состоянии \(i\) распределено экспоненциально с параметром \(\lambda_i\).
- После выхода из \(i\) новое состояние \(j\) выбирается с вероятностью
  $\[
  p_{ij} = \frac{\lambda_{ij}}{\lambda_i}.
  \]$

### Стационарное распределение
Вектор $\(\pi = (\pi_1, \pi_2, \pi_3)\)$ – стационарные вероятности, удовлетворяющие системе:
$\[
\pi Q = 0, \quad \sum \pi_i = 1,
\]$
где $\(Q\)$ – матрица интенсивностей (генератор), у которой $\(q_{ii}$ = -$\lambda_i\)$, а $\(q_{ij}$ = $\lambda_{ij}\)$ при $\(i\ne j\)$.

Эмпирическая оценка стационарного распределения получается как доля времени, проведённая в каждом состоянии при моделировании на достаточно большом интервале.

## 3. Программная реализация

### Используемые библиотеки
- **PyQt6** – графический интерфейс.
- **NumPy** – матричные вычисления.
- **SciPy** – генерация экспоненциального распределения.
- **Matplotlib** – построение графиков.
- Встроенные модули `sys`, `csv` для сохранения данных.

### Основные функции

| Функция / метод | Назначение |
|----------------|-------------|
| `build_generator_matrix(rates)` | По введённым интенсивностям строит матрицу Q (диагональ заполняется автоматически). |
| `transition_probability_matrix(q)` | Вычисляет матрицу вероятностей переходов P. |
| `stationary_distribution(q)` | Решает систему для теоретических стационарных вероятностей. |
| `simulate_ctmc(q, total_time)` | Основной алгоритм моделирования: возвращает список отрезков (начало, конец, состояние), эмпирическое распределение и другое. |
| `draw_plots()` | Визуализирует ступенчатый график смены состояний и сравнительную гистограмму. |
| `export_csv()` | Сохраняет все результаты (матрицы, статистику, историю) в CSV-файл с разделителем «;». |

### Моделирование одного шага
1. В текущем состоянии \(i\) генерируется время пребывания \( \tau \sim \text{Exp}(\lambda_i) \).
2. Если \(t + \tau \ge T_{\text{max}}\), учитывается оставшееся время и моделирование завершается.
3. Иначе копим время в счётчике `durations[i]`, продвигаем время \(t := t + \tau\).
4. Выбираем следующее состояние по распределению \(p_{i\ast}\).
5. Повторяем до достижения \(T_{\text{max}}\).
